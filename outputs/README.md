# Outputs

Generated figures, tables, logs, quality reports, and other run outputs live here.

Most generated outputs should stay out of Git.

## Current Quality Reports

- `govcn_xxgk_quality_report.csv`: quality summary for the SRDI keyword XXGK corpus.
- `govcn_xxgk_all_quality_report.csv`: quality summary for the central all-policy XXGK corpus.
- `govcn_xxgk_all_processed_v0_quality_report.csv`: QA summary for the reviewed
  processed central all-policy corpus v0.
- `manual_policy_all_keyword_srdi_quality_report.csv`: initial inspect summary
  for the manually collected national/provincial `专精特新` keyword workbook.
- `manual_policy_all_keyword_srdi_province_year_counts.csv`: province-year
  coverage counts for the same manual workbook.
- `manual_policy_srdi_processed_v0_quality_report.csv`: QA summary for the
  processed manual SRDI policy-record table and province-year intensity table.
- `manual_policy_srdi_processed_fulltext_v1_quality_report.csv`: QA summary for
  the processed manual SRDI full-text policy records.
- `manual_policy_srdi_text_mining_v0_quality_report.csv`: QA summary for the
  manual SRDI title/abstract text-mining v0 feature outputs.
- `manual_policy_srdi_text_mining_fulltext_v1_quality_report.csv`: QA summary
  for the manual SRDI full-text v1 feature outputs.
- `manual_policy_srdi_tool_dictionary_v0.csv`: transparent v0 dictionary used
  to build supply, demand, and environment policy-tool text features.
- `manual_policy_srdi_tool_dictionary_fulltext_v1.csv`: the same reviewed
  dictionary applied to title + full text.
- `manual_policy_srdi_tool_dictionary_coverage_v0.csv`: term-level coverage
  counts for reviewing the v0 dictionary.
- `manual_policy_srdi_tool_dictionary_coverage_fulltext_v1.csv`: term-level
  full-text coverage counts for the v1 dictionary run.
- `manual_policy_srdi_dictionary_revision_effect_v0.csv`: before/after effect
  of the dictionary revision based on the no-hit manual sample.
- `manual_policy_srdi_keyword_quality_check_v0.csv`: term-level keyword quality
  flags for low/high coverage, short terms, and broad-meaning terms.
- `manual_policy_srdi_no_tool_hit_records_v0.csv`: all records without a v0
  policy-tool dictionary hit.
- `manual_policy_srdi_no_tool_hit_review_sample_v0.csv`: deterministic no-hit
  review sample, 5 rows per year.
- `manual_policy_srdi_no_tool_hit_summary_v0.csv`: no-hit counts by year and
  province.
- `manual_policy_srdi_no_tool_hit_records_fulltext_v1.csv`: full-text v1 rows
  without any supply/demand/environment dictionary hit; currently only 2 rows.
- `manual_policy_srdi_no_tool_hit_summary_fulltext_v1.csv`: full-text v1 no-hit
  counts by year and province.

## Manual SRDI DeepSeek Round-1 Labeling

Tables generated by `scripts/manual_srdi_deepseek_round1_label.py` after the
live API run:

- `manual_policy_srdi_deepseek_round1_quality_report_v1.csv`: run-level
  status counts, sample-pool counts, and failure counts for DeepSeek round-1
  labeling.
- `manual_policy_srdi_deepseek_round1_run_log_v1.log`: progress, retry, cache,
  and error log for the DeepSeek round-1 labeling run.

The parsed row-level labels are written to
`data/interim/manual_policy_srdi_deepseek_labels_round1_v1.csv`; raw API
responses are cached under `data/raw/json/manual_srdi_deepseek_round1_v1/`.

QA tables generated by `notebooks/47_manual_srdi_deepseek_round1_qa.py`:

- `manual_srdi_deepseek_round1_qa_summary_v1.csv`: compact readiness summary
  for raw cache, label completeness, sample alignment, and failed-label count.
- `manual_srdi_deepseek_round1_failed_records_v1.csv`: failed parsed-label
  rows that need repair or retry before model training.
- `manual_srdi_deepseek_round1_label_distribution_v1.csv`: positive counts and
  shares for supply, demand, environment, and other labels among successful
  rows.
- `manual_srdi_deepseek_round1_pool_summary_v1.csv`: success/failure counts by
  round-1 sampling pool.
- `manual_srdi_deepseek_round1_probability_summary_v1.csv`: descriptive
  statistics for DeepSeek probability outputs.
- `manual_srdi_deepseek_round1_readiness_decision_v1.csv`: explicit readiness
  checks and recommended next action.

## Manual SRDI MacBERT Training-Data Preparation

Tables generated by `notebooks/48_manual_srdi_macbert_training_data.py`:

- `manual_srdi_macbert_training_data_quality_report_v1.csv`: readiness checks
  for the 800-row MacBERT training dataset, including split sizes and missing
  text checks.
- `manual_srdi_macbert_training_split_summary_v1.csv`: train, validation, and
  test counts by round-1 sample pool.
- `manual_srdi_macbert_training_label_balance_v1.csv`: positive and negative
  counts for supply, demand, environment, and other labels in each split.
- `manual_srdi_macbert_training_pos_weight_v1.csv`: candidate `pos_weight`
  values for `BCEWithLogitsLoss`, computed from the training split.

The row-level training dataset is written to
`data/processed/manual_policy_srdi_macbert_training_dataset_v1.csv`; JSONL split
files are written under `data/processed/manual_policy_srdi_macbert_training_v1/`.
