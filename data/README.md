# Data

Local datasets and generated data artifacts live here.

Large or generated files should stay out of Git.

## Directory Roles

- `data/raw/`: raw external inputs such as policy texts or downloaded source tables
- `data/interim/`: cleaned or merged intermediate data
- `data/processed/`: final analysis-ready datasets generated by tools
- `data/source-manifest.csv`: lightweight source register for non-test assets
- `../docs/data-dictionary.md`: datasets registry.

## Source Registration

Register auditable data artifacts in `data/source-manifest.csv`.

Required columns:

| Column | Meaning |
| --- | --- |
| `source_name` | Human-readable source or artifact group name. |
| `source_type` | Artifact type, such as `html`, `api-json`, `html-detail`, `interim-derived`, or `manual-collection`. |
| `url_or_origin` | External URL, API endpoint, upstream local file, or manual origin. |
| `access_date` | Collection or registration date. |
| `local_file` | Path relative to `data/`; grouped globs are allowed for raw crawl artifacts. |
| `generated_by` | Script, notebook, or manual process that produced the artifact. |
| `config_files` | Config files used by the run, separated with `;` when needed. |
| `upstream_files` | Local inputs used to derive the artifact, separated with `;` when needed. |
| `quality_report` | Linked quality report, if one exists. |
| `collection_status` | `raw`, `validated`, `partial`, `failed`, or `deprecated`. |
| `review_status` | `accepted`, `needs_review`, or `rejected`. |
| `record_count` | Row count for tabular artifacts; blank for grouped raw files. |
| `notes` | Short provenance note. |

Do not register every raw detail page separately. Use grouped patterns for raw
HTML/JSON and separate rows for important derived tables. See
`docs/source-manifest-guide.md` for the full convention.
