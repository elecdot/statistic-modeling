# 研究设计与变量说明

本文旨在考察“专精特新”政策对企业创新的影响，并进一步识别地方支持政策强度及政策工具结构是否会强化或削弱这一政策效果。研究设计由两个相互衔接的层次构成：第一层是企业层面的准自然实验识别，即利用专精特新“小巨人”企业认定的分批推进构造交错双重差分模型；第二层是政策文本层面的政策强度测度，即基于中央和省级专精特新相关政策文本，构建省份—年份层面的政策数量强度和政策工具强度变量，并将其并入企业面板，用于解释不同地区政策环境下政策效果的差异。

本章首先说明企业层面的识别策略，其次介绍政策文本数据的样本构造和政策工具强度的量化方法，最后给出主要变量、调节变量、控制变量和稳健性变量的定义。

## 1. 研究样本与数据结构

本文的经验分析以企业—年份面板为基本分析单位。企业层面数据用于度量企业创新产出、研发投入、企业特征和专精特新“小巨人”认定状态；政策文本层面数据用于构造企业所在省份在相应年份的专精特新政策支持强度。两类数据通过企业注册地省份和年份进行匹配。

政策文本数据来自手工收集的中央和省级专精特新相关政策记录。当前主数据源为 `data/interim/manual_policy_all_keyword_srdi_with_full_text.xlsx`，该表覆盖中央及地方政策来源中元数据包含“专精特新”的政策记录。处理后的全文政策记录表为 `data/processed/manual_policy_srdi_policy_records_fulltext_v1.csv`。本文主分析窗口为 2020—2025 年，剔除 2026 年记录后，全文政策记录共 4,475 条，其中地方政策 4,266 条，中央政策 209 条。

在地区口径上，本文将“新疆维吾尔自治区”和“新疆生产建设兵团”统一合并为“新疆”，以保证与省级面板口径一致。对于转载其他省份政策的来源站点记录，本文依据政策标题、发文主体和来源信息进行人工复核，并通过 `configs/manual_srdi_jurisdiction_overrides_v1.csv` 修正政策实际辖区。中央政策保留在行级政策记录和中央—地方描述性比较中，但由于不构成省份间政策差异，不进入省份—年份政策强度表。

最终用于 DID 衔接的政策文本变量表为 `data/processed/province_year_srdi_policy_text_variables_v1.csv`。该表以地方省份—年份为观测单位，覆盖 31 个省级单位和 2020—2025 年 6 个年份，共 186 个平衡面板观测。

## 2. 基准识别策略

本文使用交错双重差分模型识别专精特新“小巨人”认定对企业创新的影响。不同批次企业在不同年份获得“小巨人”认定，因此政策处理具有分批进入特征。设企业为 \(i\)，年份为 \(t\)，企业所在省份为 \(p(i)\)。基准模型设定为：

\[
Y_{it} = \alpha + \beta LittleGiant_{it} + \gamma' X_{it}
      + \mu_i + \lambda_t + \varepsilon_{it}
\]

其中，\(Y_{it}\) 表示企业创新结果变量，\(LittleGiant_{it}\) 为企业在年份 \(t\) 是否已获得专精特新“小巨人”认定的处理状态，\(X_{it}\) 为企业层面控制变量，\(\mu_i\) 为企业固定效应，控制企业不随时间变化的个体异质性，\(\lambda_t\) 为年份固定效应，控制宏观冲击和共同时间趋势。系数 \(\beta\) 衡量获得“小巨人”认定后企业创新表现的平均变化。

考虑到交错处理下不同批次处理组和控制组的比较结构可能影响传统双向固定效应估计，正式实证中应进一步报告适用于交错 DID 的稳健估计或事件研究结果。事件研究形式可写为：

\[
Y_{it} = \alpha + \sum_{k \neq -1} \beta_k \mathbf{1}(t - T_i = k)
      + \gamma' X_{it} + \mu_i + \lambda_t + \varepsilon_{it}
\]

其中 \(T_i\) 为企业首次获得“小巨人”认定的年份，\(k\) 表示相对认定年份。认定前的 \(\beta_k\) 用于检验平行趋势假设，认定后的 \(\beta_k\) 用于观察政策效果的动态演化。

## 3. 政策强度与政策工具结构的调节效应

本文进一步考察地方政策文本强度是否影响“小巨人”认定的创新促进效应。直观上，同样获得国家层面的“小巨人”认定，企业所在地区若具有更密集、更具体或工具结构更完整的地方支持政策，则企业可能更容易获得资金、技术服务、市场机会、融资环境和制度环境支持，从而产生更强的创新响应。

为检验这一机制，本文将省份—年份政策文本变量并入企业面板，构造如下调节效应模型：

\[
Y_{it} = \alpha + \beta LittleGiant_{it}
      + \theta \left(LittleGiant_{it} \times PolicyIntensity_{p(i),t}\right)
      + \delta PolicyIntensity_{p(i),t}
      + \gamma' X_{it} + \mu_i + \lambda_t + \varepsilon_{it}
\]

其中，\(PolicyIntensity_{p(i),t}\) 表示企业所在省份 \(p(i)\) 在年份 \(t\) 的专精特新政策强度。交互项系数 \(\theta\) 是本文关注的核心参数之一，反映地方政策强度是否会调节“小巨人”认定对企业创新的影响。如果 \(\theta > 0\)，说明地方专精特新政策支持越强，获得“小巨人”认定后的创新提升越明显。

在政策工具结构分析中，本文将 \(PolicyIntensity_{p,t}\) 分别替换为供给型、需求型和环境型政策工具强度：

\[
Y_{it} = \alpha + \beta LittleGiant_{it}
      + \theta_s \left(LittleGiant_{it} \times SupplyIntensity_{p(i),t}\right)
      + \theta_d \left(LittleGiant_{it} \times DemandIntensity_{p(i),t}\right)
      + \theta_e \left(LittleGiant_{it} \times EnvironmentIntensity_{p(i),t}\right)
      + \gamma' X_{it} + \mu_i + \lambda_t + \varepsilon_{it}
\]

其中，\(\theta_s\)、\(\theta_d\) 和 \(\theta_e\) 分别刻画供给型、需求型和环境型政策工具对“小巨人”认定效果的调节作用。

## 4. 政策文本分类与政策工具强度构造

### 4.1 政策工具分类框架

本文将专精特新政策工具划分为供给型、需求型和环境型三类。供给型工具主要指政府直接提供资源和服务，例如财政奖励、补贴补助、人才培训、公共服务平台、技术改造支持、研发支持和空间要素保障等。需求型工具主要指政府通过创造、扩大或引导市场需求支持企业发展，例如政府采购、首台套推广、示范应用、应用场景开放、供需对接、展会推广、市场开拓和产业链对接等。环境型工具主要指政府改善制度、金融、知识产权和营商环境，例如融资担保、贷款贴息、上市培育、知识产权保护、标准制定、认定评价、信用体系和服务机制建设等。

由于同一篇政策文件可能同时包含多种政策工具，本文不采用互斥式单标签分类，而采用多标签分类方法。对每篇政策文件 \(i\)，模型分别输出四个概率：

\[
p^{supply}_i,\quad p^{demand}_i,\quad p^{environment}_i,\quad p^{other}_i
\]

前三个概率表示政策文件包含相应政策工具的概率，不要求三者互斥，也不要求概率之和为 1。\(p^{other}_i\) 表示该文本更接近纯程序性通知、公示、名单、新闻、政策解读或其他不适合进入实质性政策工具分析的边界文本。

### 4.2 标注与模型训练

本文首先基于全文关键词规则构造供给型、需求型、环境型和其他类候选池，并按候选池分层抽取 800 篇政策文本进行首轮标注。标注阶段使用 DeepSeek 生成多标签银标，每篇文本输出供给型、需求型、环境型和其他类概率，并保留支持判断的文本证据。随后，本文以 800 篇银标样本训练中文 MacBERT 多标签分类模型，并将训练后的模型应用于 4,475 篇全文政策记录。

当前 round-1 MacBERT 模型在测试集上的 micro-F1 为 0.853，macro-F1 为 0.777。分标签看，供给型、需求型和环境型三类主要政策工具的 F1 分别约为 0.852、0.818 和 0.910，表现明显优于边界性质的 `other` 类。基于这一结果，本文将三类主要工具概率用于政策强度构造，而将 `other` 定位为审计和过滤变量，不作为实质性政策工具维度解释。

### 4.3 概率化政策工具强度

设 \(D_{p,t}\) 表示省份 \(p\) 在年份 \(t\) 的专精特新相关政策文件集合。本文将某类政策工具强度定义为该省该年所有政策文件中该类工具概率的加总：

\[
SupplyIntensity_{p,t}
= \sum_{i \in D_{p,t}} p^{supply}_i
\]

\[
DemandIntensity_{p,t}
= \sum_{i \in D_{p,t}} p^{demand}_i
\]

\[
EnvironmentIntensity_{p,t}
= \sum_{i \in D_{p,t}} p^{environment}_i
\]

该指标的含义不是财政支出规模或真实执行力度，而是“基于多标签分类概率的政策工具预期覆盖强度”。例如，\(SupplyIntensity_{p,t}\) 可理解为省份 \(p\) 在年份 \(t\) 的政策文本中供给型工具出现的期望规模。与硬标签计数相比，概率加总保留了模型置信度，并能更平滑地刻画政策文本中混合政策工具的强弱差异。

本文同时构造总政策工具强度：

\[
TotalToolIntensity_{p,t}
= SupplyIntensity_{p,t}
 + DemandIntensity_{p,t}
 + EnvironmentIntensity_{p,t}
\]

以及政策工具结构占比：

\[
SupplyShare_{p,t}
= \frac{SupplyIntensity_{p,t}}
{SupplyIntensity_{p,t} + DemandIntensity_{p,t} + EnvironmentIntensity_{p,t}}
\]

需求型和环境型占比同理构造。绝对强度回答“该省该年某类工具多不多”，结构占比回答“该省该年政策工具组合是否更偏向某一类型”。

需要强调的是，概率加总强度基于两个关键假设。第一，模型输出概率可近似解释为文本包含某类政策工具的概率。第二，每篇政策文件在基准设定中具有相同基础权重。因此，该指标应被称为“政策工具概率化强度”或“政策工具预期覆盖强度”，而非未经限定的真实政策执行强度。若要进一步接近执行力度，可在后续稳健性检验中引入政策层级、文件类型、联合发文或文本长度等权重。

## 5. 主要变量定义

### 5.1 被解释变量：企业创新

本文的被解释变量为企业创新表现。根据后续企业面板数据可得性，可从以下维度度量：

1. 专利产出。包括企业年度专利申请数量、授权数量、发明专利申请数量或发明专利授权数量。由于专利数量通常右偏，实证中可使用 \(\log(1 + PatentCount_{it})\)。
2. 研发投入。包括研发费用、研发投入强度或研发人员数量等。如果数据可得，研发投入可用于检验“小巨人”认定是否提升企业创新投入。
3. 创新质量。可使用发明专利占比、被引用专利或高价值专利指标作为补充结果变量。

正式论文中应根据企业数据最终可得字段确定主结果变量。若专利数据最完整，建议以专利申请或发明专利申请作为主被解释变量，以授权专利、研发投入或创新质量指标作为稳健性和机制结果。

### 5.2 核心解释变量：专精特新“小巨人”认定

核心处理变量为 \(LittleGiant_{it}\)，表示企业 \(i\) 在年份 \(t\) 是否已经获得专精特新“小巨人”认定。企业首次获得认定的年份记为 \(T_i\)。若 \(t \geq T_i\)，则 \(LittleGiant_{it}=1\)；否则为 0。未获得认定的企业在所有年份取 0。

在事件研究中，本文进一步构造相对认定年份 \(EventTime_{it}=t-T_i\)，用于检验认定前趋势和认定后动态效应。

### 5.3 基础政策强度变量

基础政策强度变量用于刻画省份—年份层面专精特新相关政策出台的活跃程度，来自 `data/processed/province_year_srdi_policy_text_variables_v1.csv`：

| 变量 | 定义 | 用途 |
| --- | --- | --- |
| `srdi_policy_count` | 省份—年份内专精特新相关政策数量 | 政策数量强度；可作为调节变量或控制变量 |
| `srdi_policy_count_log` | \(\log(srdi\_policy\_count+1)\) | 缓解政策数量右偏的替代指标 |

其中，`srdi_policy_count` 反映地方政府围绕专精特新企业出台政策的活跃程度。由于该指标透明、稳定，本文建议将其作为基准政策支持强度或政策数量控制变量。

### 5.4 主政策工具强度变量

本文的主政策工具变量为连续 MacBERT 概率加总强度：

| 变量 | 定义 | 理论含义 |
| --- | --- | --- |
| `srdi_supply_intensity` | 省份—年份内 \(\sum p^{supply}_i\) | 供给型政策工具预期覆盖强度 |
| `srdi_demand_intensity` | 省份—年份内 \(\sum p^{demand}_i\) | 需求型政策工具预期覆盖强度 |
| `srdi_environment_intensity` | 省份—年份内 \(\sum p^{environment}_i\) | 环境型政策工具预期覆盖强度 |
| `srdi_total_tool_intensity` | 三类工具强度之和 | 总政策工具文本强度 |

在主模型中，本文优先使用 `srdi_supply_intensity`、`srdi_demand_intensity` 和 `srdi_environment_intensity` 分别检验三类政策工具的调节作用。`srdi_total_tool_intensity` 可用于描述总体政策工具强度，但不应替代分工具变量，因为不同政策工具可能具有不同作用机制。

### 5.5 政策工具结构变量

为刻画政策工具组合，本文进一步使用三类概率占比：

| 变量 | 定义 |
| --- | --- |
| `srdi_supply_probability_share` | 供给型概率强度占三类工具概率强度之比 |
| `srdi_demand_probability_share` | 需求型概率强度占三类工具概率强度之比 |
| `srdi_environment_probability_share` | 环境型概率强度占三类工具概率强度之比 |

这些变量衡量的不是政策数量多寡，而是政策工具配置结构。例如，在政策数量相近的两个省份中，需求型占比更高的省份可能更强调市场开拓、场景应用和采购推广。

### 5.6 审计与过滤变量

`other` 类用于识别纯程序性通知、公示、名单、新闻、政策解读或其他非实质政策工具文本。由于 `other` 类不是政策工具类型，且当前模型中其预测性能低于三类主标签，本文不将其作为政策工具强度解释，而仅用于审计和稳健性过滤。

| 变量 | 定义 | 用途 |
| --- | --- | --- |
| `srdi_valid_tool_policy_count` | 通过 hard-label 审计规则保留的有效政策工具文本数量 | 数据质量审计 |
| `srdi_valid_tool_policy_share` | 有效政策工具文本占比 | 数据质量审计 |
| `srdi_other_exclusion_count` | hard `other` 文本数量 | 边界文本审计 |
| `srdi_supply_intensity_filtered` | 排除 hard `other` 后的供给型概率强度 | 稳健性变量 |
| `srdi_demand_intensity_filtered` | 排除 hard `other` 后的需求型概率强度 | 稳健性变量 |
| `srdi_environment_intensity_filtered` | 排除 hard `other` 后的环境型概率强度 | 稳健性变量 |

此外，也可构造基于 \(1-p^{other}_i\) 的折减强度：

\[
WeightedSupplyIntensity_{p,t}
= \sum_{i \in D_{p,t}} (1-p^{other}_i) p^{supply}_i
\]

需求型和环境型同理。该版本更强调对边界文本的概率折减，可作为对主概率加总指标的稳健性检验。

### 5.7 稳健性变量

为验证结论不依赖于概率加总形式，本文保留两类稳健性变量。

第一类是 MacBERT 硬标签计数：

| 变量 | 定义 |
| --- | --- |
| `srdi_supply_hard_label_count` | \(p^{supply}_i \geq 0.5\) 的政策数量 |
| `srdi_demand_hard_label_count` | \(p^{demand}_i \geq 0.5\) 的政策数量 |
| `srdi_environment_hard_label_count` | \(p^{environment}_i \geq 0.5\) 的政策数量 |

第二类是全文词典计数：

| 变量 | 定义 |
| --- | --- |
| `dict_supply_policy_count` | 全文词典命中供给型词项的政策数量 |
| `dict_demand_policy_count` | 全文词典命中需求型词项的政策数量 |
| `dict_environment_policy_count` | 全文词典命中环境型词项的政策数量 |

硬标签计数检验结果是否依赖概率强弱，词典计数检验结果是否依赖监督模型。若主结果在概率加总、硬标签计数和透明词典计数下方向一致，则可以增强政策文本变量的可信度。

## 6. 控制变量与固定效应

企业层面控制变量应根据最终企业数据库可得性确定，通常包括企业规模、企业年龄、资产负债率、盈利能力、所有制、行业、地区和上市状态等。若使用上市公司或规上工业企业数据，还可加入营业收入、总资产、员工人数、资本密集度、现金流、研发基础等变量。

固定效应方面，基准模型建议至少包含企业固定效应和年份固定效应。企业固定效应控制企业不随时间变化的创新能力、管理质量和行业定位，年份固定效应控制宏观经济、全国性产业政策和整体创新环境变化。根据数据结构，还可加入行业—年份固定效应或省份线性趋势，以控制行业周期和地区长期趋势。

标准误应至少在企业层面聚类。若政策强度变量在省份—年份层面变化，稳健性检验中也可考虑省份层面聚类或双向聚类，以回应同一地区企业误差项相关的问题。

## 7. 稳健性检验设计

本文的稳健性检验可围绕识别策略和政策文本变量两条线展开。

第一，针对 DID 识别，进行平行趋势检验和事件研究，观察处理前各期系数是否接近 0；进行不同窗口设定、不同控制组设定和不同固定效应设定的稳健性检验。

第二，针对政策文本变量，使用替代测度检验结论稳健性。具体包括：

1. 将主概率加总强度替换为排除 hard `other` 后的 filtered intensity。
2. 将连续概率强度替换为 hard-label count。
3. 将 MacBERT 测度替换为全文词典 count 或 share。
4. 将政策工具绝对强度替换为政策工具结构占比。
5. 将 `srdi_policy_count` 或 `srdi_policy_count_log` 加入模型，检验工具变量效果是否仅由政策数量驱动。

第三，针对边界文本，复核 `outputs/manual_srdi_macbert_full_corpus_boundary_samples_v1.csv` 中的高 `p_other`、低置信度、需求阈值附近和 MacBERT/词典冲突样本。该复核主要用于论文最终冻结前的质量审计，而不是重新定义主变量。

## 8. 变量口径总结

本文主变量口径可概括如下：

1. 企业层面主处理变量为是否已获得专精特新“小巨人”认定。
2. 政策数量强度以省份—年份专精特新相关政策数量及其对数形式衡量。
3. 政策工具强度以 MacBERT 多标签模型输出概率的省份—年份加总衡量，分别对应供给型、需求型和环境型工具。
4. 概率加总强度应解释为“政策工具预期覆盖强度”或“概率化政策工具强度 proxy”，不直接等同于财政金额、执行力度或法律约束力。
5. `other` 类用于边界文本审计和稳健性过滤，不作为实质性政策工具。
6. hard-label 计数和全文词典计数作为稳健性变量，帮助检验主结果是否依赖模型概率形式。

据此，本文能够在企业层面识别“小巨人”认定的创新效应，并进一步考察地方政策支持强度和政策工具结构是否影响这一效应，为理解专精特新政策的地区差异和工具机制提供文本量化证据。
